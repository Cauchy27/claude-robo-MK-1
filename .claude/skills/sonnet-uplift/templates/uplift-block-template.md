# 品質ゲート（sonnet-uplift）埋め込みテンプレート

対象エージェント定義の「判断基準」（相当セクション）の直後に、以下のブロックを追記する。
プレースホルダー `{...}` は Phase 1 の設計内容で置換すること。全体で30-50行以内に収める。

設計原則: [../reference.md](../reference.md) セクション3・4 を参照。

---

## テンプレート（コピー用）

```markdown
## 品質ゲート（sonnet-uplift）

### 完了条件チェックリスト

出力を返す前に、以下を全て満たすことを確認する:

- [ ] {二値判定可能な条件1}
- [ ] {二値判定可能な条件2}
- [ ] {二値判定可能な条件3}
- [ ] {二値判定可能な条件4}
- [ ] {二値判定可能な条件5}

### 自己検証ループ

1. 成果物のドラフトを作成する
2. 完了条件チェックリストと照合する
3. NG 項目があれば修正して再照合する（最大2周）
4. 2周で全項目 OK にならない場合はエスカレーションする

### エスカレーション基準

以下のいずれかに該当する場合、自力で完結させず、出力の冒頭に次の形式で返す:

- 自己検証ループ2周でチェックリストが収束しない
- チェックリストにないトレードオフ判断が必要になった
- 不可逆操作・対外公開・機密・法務/人事に関わる判断が必要になった
- {タスク固有のエスカレーション条件（任意・0-2個）}

```
ESCALATION: {理由を1文}
完了済み: {完了した部分}
残作業: {残っている判断・作業}
判断材料: {親が判断するために必要な情報}
```

### 既知の失敗パターン

- {このエージェントで実際に起きた・起きやすい失敗（1-3個）}
```

## 記入例（report-writer の場合）

```markdown
## 品質ゲート（sonnet-uplift）

### 完了条件チェックリスト

出力を返す前に、以下を全て満たすことを確認する:

- [ ] 結論（要約）が文書の冒頭セクションにある
- [ ] 全ての数値・データに出典が明記されている
- [ ] アクションアイテムに担当者と期限が付いている
- [ ] 見出し階層が飛び級していない（h2 の次は h3）
- [ ] 依頼時に指定された読者層に合わない専門用語が説明なしで使われていない

### 自己検証ループ

1. 成果物のドラフトを作成する
2. 完了条件チェックリストと照合する
3. NG 項目があれば修正して再照合する（最大2周）
4. 2周で全項目 OK にならない場合はエスカレーションする

### エスカレーション基準

以下のいずれかに該当する場合、自力で完結させず、出力の冒頭に次の形式で返す:

- 自己検証ループ2周でチェックリストが収束しない
- チェックリストにないトレードオフ判断が必要になった
- 不可逆操作・対外公開・機密・法務/人事に関わる判断が必要になった
- 元データ同士が矛盾しており、どちらを正とするか判断できない

```
ESCALATION: {理由を1文}
完了済み: {完了した部分}
残作業: {残っている判断・作業}
判断材料: {親が判断するために必要な情報}
```

### 既知の失敗パターン

- 元資料にない数値を推定で補完してしまう（出典のない数値は「不明」と書く）
- 長文レポートで冒頭要約を省略する
```

## 適用時の注意

| # | 注意 | 理由 |
|---|------|------|
| 1 | frontmatter は変更しない | model の変更は agent-model-tuning のフローで別途実施する |
| 2 | 既存セクションを書き換えない | ペルソナ定義（Who）とテンプレート準拠を崩さないため |
| 3 | ブロックは1エージェント1個まで | 重複埋め込みはコンテキスト税の二重払いになる |
| 4 | 記入例の丸写し禁止 | チェックリストは対象エージェントの失敗モードから設計する |
